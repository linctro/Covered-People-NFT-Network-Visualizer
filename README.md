# Covered-People-NFT-Network-Visualizer# Covered People NFT Network Visualizer 仕様書

## プロジェクト概要

### 目的
カバードピープル（2022年発足のweb3コミュニティ）が発行したNFTの取引履歴を木の年輪をモチーフとした視覚的なネットワーク図で表現するwebアプリケーションの開発。

### コンセプト
- 中心から放射状に広がる年輪構造
- NFTの転送履歴を星座やアメーバのような有機的な線で表現
- 時系列による層構造（2022年中心、外側に向かって年数が進む）

## NFT情報

### 発行NFT
1. **ジェネシス**: 200点（黄色 #FFD700で表示）
2. **ジェネラティブ**: 3,333点（緑 #00FF7Fで表示）
3. **総発行数**: 3,533点

### 取引傾向
- 2022年〜2023年: 比較的活発な取引
- 現在（2024年以降）: 月10回程度の取引頻度

## 技術仕様

### フロントエンド
- **HTML5 Canvas**: メインビジュアライゼーション
- **CSS3**: レスポンシブデザイン、アニメーション
- **Vanilla JavaScript**: インタラクション制御

### バックエンド（将来実装）
- **Node.js + Express**: データ処理
- **ブロックチェーンAPI連携**: 実際の取引データ取得

### データソース
- ブロックチェーンの取引記録から直接取得
- JSON形式でのデータ交換

## UI/UX設計

### レイアウト構成
```
[メインキャンバス（年輪ネットワーク図）] [サイドバー（統計・凡例）]
[コントロール（ズーム等）]              [詳細パネル（選択時表示）]
```

### 色彩設計
- **背景**: 深い紺色グラデーション（#0a0a0f → #1a1a2e）
- **ジェネシスNFT**: 金色（#FFD700）
- **ジェネラティブNFT**: 緑色（#00FF7F）
- **発行者ノード**: 白色（#ffffff）
- **接続線**: 半透明青（rgba(100, 200, 255, 0.3)）
- **年輪**: 半透明白（rgba(255, 255, 255, 0.1)）

## 機能仕様

### 基本機能
1. **年輪レイヤー表示**
   - 2022年: 中心円
   - 2023年〜2032年: 外側に向かって層を形成
   - 各層は同心円状に配置

2. **ノード表示**
   - 発行者: 中心点（白色、半径8px）
   - NFT保有者: 各年輪層内に配置（半径4px）
   - 色分け: ジェネシス（黄）、ジェネラティブ（緑）

3. **エッジ表示**
   - NFT転送を線で表現
   - 転送元→転送先への方向性
   - 有機的なランダム配置

### インタラクション機能
1. **ナビゲーション**
   - マウスドラッグ: パンニング
   - マウスホイール: ズームイン/アウト
   - コントロールボタン: ズーム操作、ビューリセット

2. **選択・詳細表示**
   - ノードクリック: 詳細情報表示
   - 選択時のハイライト表示
   - 詳細パネルでの情報表示

3. **フィルタリング（将来実装）**
   - NFTタイプ別表示切替
   - 年代別フィルター
   - 特定NFTの取引履歴追跡

### 表示情報

#### サイドバー統計
- 総NFT数: 3,533
- 総取引数: 動的カウント
- アクティブ年数
- 年別取引数内訳
- 凡例（色分け説明）

#### 詳細パネル
- NFT ID
- 発行年
- NFTタイプ（ジェネシス/ジェネラティブ）
- トークンID
- 転送回数
- 現在の保有者情報

## データ構造

### ノードデータ
```javascript
{
  id: string,           // 一意識別子
  x: number,           // X座標
  y: number,           // Y座標
  type: 'issuer' | 'holder',  // ノードタイプ
  year: number,        // 年
  radius: number,      // 表示半径
  nftType: 'genesis' | 'generative' | null,
  tokenId: number,     // トークンID
  transfers: number,   // 転送回数
  address: string      // ウォレットアドレス（将来実装）
}
```

### エッジデータ
```javascript
{
  source: string,      // 転送元ノードID
  target: string,      // 転送先ノードID
  year: number,        // 転送年
  blockNumber: number, // ブロック番号（将来実装）
  timestamp: number    // タイムスタンプ（将来実装）
}
```
